# คู่มือการออกแบบและการปรับแต่ง UI (UI & Dumb UI Concept)

ปัญหาคลาสสิกของการทำระบบ Inventory คือ **"UI มักจะเข้าไปยุ่งกับ Logic เสมอ"** เช่น เมื่อผู้เล่นคลิกขวาที่ยาเพื่อกินยา UI มักจะเป็นคนเขียนโค้ดเช็คว่า "ช่องนี้มียาไหม?" "ลดจำนวนลง 1" และ "เพิ่ม HP ผู้เล่น"

ผลลัพธ์คือ: เมื่อคุณเปลี่ยน UI หรือย้ายไปทำเกมมือถือ โค้ดทั้งหมดจะพัง!

Universal Inventory แก้ปัญหานี้ด้วยแนวคิดที่เรียกว่า **"Dumb UI" (UI ที่โง่ที่สุด)**

---

## 🧠 Dumb UI ทำงานอย่างไร?

ในระบบของเรา `InventoryUI` และ `EquipmentUI` จะทำหน้าที่แค่ 2 อย่างเท่านั้น:
1. **วาดภาพตามคำสั่ง (Draw on Signal):** เมื่อ `InventoryComponent` มีการเปลี่ยนแปลง มันจะตะโกนบอก (Emit Signal) ว่า `inventory_changed` UI จะได้ยินแล้วล้างหน้าจอเพื่อวาดรูปไอเทมใหม่
2. **เป็นกระบอกเสียง (Relay User Input):** เมื่อผู้เล่นลากเมาส์ (Drag & Drop) หรือคลิกขวา UI จะไม่ตรวจสอบเงื่อนไขใดๆ ทั้งสิ้น มันจะโยนคำสั่งไปให้ **API Layer** จัดการ

ด้วยโครงสร้างนี้ คุณสามารถลบ UI ของเราทิ้งทั้งหมด แล้วเขียน UI 3D VR สวมแว่นตาขึ้นมาใหม่ ระบบ Core ก็จะยังทำงานได้ตามปกติ 100%

---

## 🔌 การผูก UI เข้ากับข้อมูล (Binding)

สคริปต์ UI ของปลั๊กอินได้รับการออกแบบมาให้ใช้ซ้ำ (Reusable) ได้กับทุกอย่าง ไม่ว่าจะเป็นกระเป๋าผู้เล่น กล่องสมบัติ หรือศพมอนสเตอร์ 

คุณเพียงแค่ต้องส่งตัวแปร (Reference) ของ Core Component ไปให้มันตอนเริ่มเกม:

```gdscript
# ตัวอย่างโค้ดใน Player.gd หรือ UIManager.gd
@onready var inventory_ui = $CanvasLayer/InventoryUI
@onready var player_inventory = $Player/InventoryComponent

func _ready():
    # สั่งให้ UI รู้จักกับกระเป๋าของผู้เล่น
    inventory_ui.set_inventory(player_inventory)
```
ทันทีที่เรียก `set_inventory()` ตัว UI จะ:
1. ลบช่องเก่าทิ้งทั้งหมด
2. สร้างช่องขึ้นมาใหม่ตาม `max_slots` ที่ตั้งไว้ใน Component
3. ทำการ `connect()` เข้ากับ Signal แบบอัตโนมัติ

## 🎨 การปรับแต่งความสวยงาม (Theming)

ปลั๊กอินมาพร้อมกับ Prefab UI ที่ทำงานได้สมบูรณ์แบบ คุณสามารถปรับแต่งหน้าตาได้อย่างอิสระโดยไม่ต้องกลัวโค้ดพัง

### `InventorySlotUI.tscn` (ช่องตารางแต่ละช่อง)
- **TextureRect (พื้นหลัง):** เปลี่ยนรูปภาพเป็นสไตล์ของเกมคุณได้
- **Label (จำนวน):** เปลี่ยนฟอนต์ ขนาด และสี
- **ColorRect (Highlight):** เป็นแถบสีที่จะแสดงขึ้นมาเวลาเอาเมาส์ไปชี้ (Hover) คุณสามารถเปลี่ยนสี หรือเปลี่ยนเป็นอนิเมชั่นได้
- **ProgressBar (หลอดต่างๆ):** สำหรับไอเทมที่มี `PerishableModule` หรือหลอดความทนทาน UI จะวาด ProgressBar ซ้อนทับให้อัตโนมัติ

## 🏗️ `item_database.gd` (เครื่องมือฝั่ง Editor)

นอกจาก UI ในเกม (Runtime) แล้ว ปลั๊กอินเรายังมี UI พิเศษที่ทำงานใน Godot Editor เรียกว่า **Item Database Editor** (อยู่ที่ Bottom Panel)

นี่คือสุดยอดเครื่องมือสำหรับนักออกแบบเกม (Game Designer) โดยเฉพาะ:
- ใช้วาดหน้าจอ (Inspector) แบบมีลวดลาย
- รองรับการแสดงผล **"รายชื่อโมดูลทั้งหมด"** ที่ไอเทมนั้นมี
- หากคุณเขียน Custom Module ของตัวเองขึ้นมา หน้าจอ Database นี้ก็จะปรับตัวและแสดงผลฟิลด์ที่คุณ `@export` ไว้โดยอัตโนมัติ!

*(ระบบนี้เป็นเครื่องพิสูจน์ถึงความยืดหยุ่นของ Data-Driven คุณมีหน้าที่แค่สร้าง Module ส่วน UI จะจัดการวาดฟอร์มให้กรอกข้อมูลเอง)*
